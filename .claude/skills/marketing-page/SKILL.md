# Focus MÃ¡s marketing page

`src/marketing/` is the public landing page. It renders when `authStatus === 'signed-out'`,
at any path (`src/App.tsx`). A stranger following a shared link always lands here.

Its job is **impression, not conversion**. There is no signup funnel, no pricing, no email
capture, no SEO work. The audience is recruiters, peer engineers, and friends â€” most will
never sign in. Do not add social-proof sections, testimonials, or PAS/AIDA/StoryBrand copy
structures. This page has zero users and no conversion goal; conversion-optimization advice
actively makes it worse.

## Hard rules

1. **Never import a writer.** No `lib/sessions`, `lib/plannerDays`, `lib/firebase`, or any
   Firestore call. No `useAppStore` writes. The only permitted side effect is
   `signInWithGoogle()` from `lib/auth`, triggered by an explicit click.
2. **Never fake a `userId`.** `Timer.tsx` gates its persistence on `userId`; faking one arms
   `createSessionRef`, `appendFocusSnapshot`, and `completeSession`. A marketing page must
   never write to the database.
3. **Reuse presentational components, not containers.** `TimerDisplay`, `PostSessionSurvey`,
   and `CalendarDayCell` are pure and prop-driven â€” use them directly. Do **not** add
   marketing-only props to `Timer`, `ContributionCalendar`, `MasteryStage`, or `StreakBar`;
   those carry app behavior (lazy month expansion, scroll-to-today, store reads) the page
   does not want.
4. **Derive, never hardcode.** The stage ladder comes from `getStageLadder()`
   (`lib/beltColors.ts`). Demo sessions are generated by running the real `mastery.ts`
   progression rules. If the app's rules change, the page must change with them. Never
   hand-place a belt color or a stage number.
5. **Respect motion and battery.** The hero loop pauses on
   `document.visibilityState === 'hidden'` and shows a static end-state under
   `prefers-reduced-motion: reduce`, matching the convention in `src/index.css`.
6. **Scoped scroll only.** `.marketing-shell` is its own scroll container nested inside the
   `#root` overflow lock. Never relax `html`/`body`/`#root` overflow to make the page scroll.

## Section spine

Fixed order. Each section answers the question the previous one raises.

1. **Hero** â€” thesis + the compressed autoplay demo loop.
2. **Ladder** â€” `getStageLadder()`, cream 25 min â†’ espresso 90 min.
3. **Calendar** â€” an illustrated year, explicitly captioned as an illustration.
4. **Close** â€” sign in with Google.

Ladder **must** precede calendar: the calendar's belt colors are meaningless until the
ladder has taught the reader that cream is 25 and espresso is 90.

## The demo loop

Compressed clock, roughly twelve seconds, then reset. Counts 25:00 down fast, hits zero,
slides up the real `PostSessionSurvey`, fills a calendar cell, ticks the streak bar 3/5 â†’
4/5, resets. No start button â€” a real session is 25 minutes and nobody waits.

## Voice

Plain and declarative. Short sentences, no metaphor, no exclamation marks, no second-person
coaching. It matches the app's own copy â€” `Timer.tsx` says *"Stopping early discards this
session."* Write like that.

The page currently stays silent on the fact that the distraction survey is self-reported.
That was a deliberate choice. If asked to address it, name it as a design decision rather
than defending it: the reward for lying is a longer timer, which is harder.

## Style

Use the existing glass surface classes (`.glass-card`, `.glass-btn`, `.glass-panel`) and the
`liquidglass-tailwind` utilities (`rounded-glass`, `rounded-glass-lg`). See the
`liquid-glass-css-skill` and `liquidglass-tailwind` skills. The wallpaper
(`.app-shell::before`) is `position: fixed`, so it parallaxes behind the scrolling content
for free. `CursorLight` already renders in the signed-out tree.
